# Arx Notes `:crypto`

> 🇬🇧 [English version](README.md)

Открытое крипто-ядро приложения Arx Notes. Самодостаточный Android-модуль без
зависимостей от UI/данных приложения — его можно читать, аудировать и переиспользовать
отдельно.

Лицензия: **Apache-2.0** (см. [LICENSE](LICENSE)). Модель угроз: [THREAT_MODEL.md](THREAT_MODEL.md).

## Статус аудита (честно)

Модуль покрыт собственным тест-набором (73 проверки, включая эталонные векторы RFC 5869 и
RFC 9106) и внутренней самооценкой. **Независимый внешний аудит НЕ проводился.** Ничего здесь
не следует воспринимать как стороннюю проверку. Независимое ревью приветствуется — карта для
аудитора в [AUDIT-SCOPE.md](AUDIT-SCOPE.md).

## Состав

| Компонент | Назначение | Примитивы |
|---|---|---|
| `MasterKeyManager` | Корень доверия: случайный 256-бит мастер-секрет, обёрнутый ключом Android Keystore. Раздаёт рабочие подключи. | AES-256-GCM (Keystore), HKDF-SHA256 |
| `Hkdf` | Доменное разделение ключей из мастера (RFC 5869). | HMAC-SHA256 |
| `AeadBlob` | Общий примитив самоописывающего AES-GCM-блоба `[ver][IV][ct+tag]`. | AES-256-GCM |
| `AudioCrypto` | Шифрование голосовых заметок. | AES-256-GCM, ключ = HKDF(`audio`) |
| `BackupCrypto` | Парольные файлы бэкапа. | Argon2id + AES-256-GCM |
| `PinHasher` | Одностороннее хеширование PIN блокировки (проверка, не расшифровка). | Argon2id (соль + хеш) |

## Иерархия ключей

```
Android Keystore (не извлекаемый; аппаратный/TEE, где даёт платформа)
        │  обёртка AES-GCM
        ▼
   Мастер-секрет (32 байта случайных, на диске только в зашифрованном блобе)
        │  HKDF-SHA256(info=…)
        ├── info "app.safenote/db/v1"     → ключ SQLCipher
        └── info "app.safenote/audio/v1"  → ключ AudioCrypto

Пароль пользователя ──Argon2id(salt, params)──► ключ файла бэкапа   (не связан с мастером)
PIN блокировки ──Argon2id(salt)──► хеш для проверки                 (односторонний, не ключ)
```

Один секрет не используется напрямую в двух криптосистемах: рабочие ключи —
независимые HKDF-подключи с доменными метками.

> **Метки `app.safenote/*` заморожены (это не забытый бренд).** Префикс `app.safenote/…` в доменных
> метках HKDF (и alias `safenote_master_key` в Keystore) — это **идентификатор обратной
> совместимости, а не название бренда**: он намеренно *не* переименован в `arxnotes`. Смена этих
> значений выведет другие ключи / оставит без ключа-обёртки и сделает уже зашифрованные данные
> нечитаемыми; менять их можно только через версионную миграцию (напр. `/v2`).

## Принципы

- **Только аутентифицированное шифрование** (AES-GCM / Argon2id+GCM). Подмена/обрезка → провал расшифровки.
- **Случайность из `SecureRandom`** для всех ключей, солей и nonce.
- **Безопасные дефолты, без переключателей.** Шифрование — свойство хранилища, не настройка.
- **Без сети и Google-сервисов** в самих примитивах (Argon2 — BouncyCastle, чистая Java; никаких внешних крипто-библиотек).
- **Версионированные форматы** с параметрами KDF в заголовке — усиление без поломки старых файлов.

## Форматы

**Бэкап** (`BackupCrypto`):
```
magic "SNBK" | ver=2 | kdf=1(Argon2id) | mem KiB(4) | iters(1) | par(1) | salt(32) | nonce(12) | ciphertext+tag
```
**AES-GCM-блоб** (`AeadBlob` — обёртка мастер-секрета и аудио): `ver=1 | IV(12) | ciphertext+tag`.

## Тесты

Набор сгруппирован по проверяемым свойствам (round-trip / целостность / уникальность
nonce / KDF-PIN / гигиена), включает эталонные векторы RFC 5869 и RFC 9106. Запуск:

```bash
./gradlew :crypto:test                  # модульные (JVM, без устройства)
./gradlew :crypto:connectedAndroidTest  # инструментальные (нужно устройство/эмулятор)
```

Полная карта покрытия, эталонные векторы, что НЕ покрыто и почему, а также ручные
проверки — в [TESTING.md](TESTING.md). Машинные отчёты пишутся в `crypto/build/` и не
коммитятся (исключены в `.gitignore`) — результат воспроизводится повторным запуском.
